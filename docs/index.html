Setup without the dev ritual.

Invite YapHub, approve the requested permissions, then run one command in Discord. No cloning, no self-hosting, no token wrangling, no README archaeology.

Add YapHub

Use the Add to Discord button and approve the requested permissions.

Run setup

Use /yap setup and select where temporary rooms should be created.

Join the lobby

Members join the configured lobby and get moved into their temporary room.

Let it clean up

When the room empties, YapHub deletes it.

Built for Discord communities.

Gaming servers, leagues, friend groups, creator communities — anywhere voice channels multiply like gremlins after midnight.

/yap config

Permissions, not admin keys.

YapHub asks for voice and channel-management permissions so it can create temporary rooms, move members, and remove rooms later. It should not need full Administrator access.

Built narrow on purpose.

YapHub is not trying to be a mega-bot. It focuses on temporary voice channels, category-scoped join lobbies, and keeping Discord servers from turning into permanent voice-channel soup.
